# User Stories

## Engineer

- As an engineer, I want a short ranked feed so I can quickly see what may affect my tools, systems, or career.
- As an engineer, I want each item to explain why engineers should care so I can decide whether to change a workflow, learn something new, or ignore it.
- As an engineer, I want duplicate coverage merged into one main item so the dashboard does not waste my time.
- As an engineer, I want source links and confidence signals so I can verify important updates quickly.

## Student

- As a student, I want simple summaries so I can understand important industry changes without already knowing the jargon.
- As a student, I want suggested actions so I know what to learn, try, or research next.
- As a student, I want the feed to favor a few important updates over a long noisy stream so I do not feel overwhelmed.
- As a student, I want each item to say who should care so I can judge whether it matters for internships or project work.

## Technical Founder

- As a technical founder, I want startup, tooling, infrastructure, and company-move signals in one place so I can spot risks and opportunities early.
- As a technical founder, I want source-aware confidence so I can distinguish official changes from market noise.
- As a technical founder, I want engineering-focused summaries rather than generic business coverage so I can assess technical impact fast.
- As a technical founder, I want refresh-based discovery instead of constant feed churn so I can check the product intentionally.

## Integration Developer

- As an integration developer, I want a stable `FeedResponse` contract so backend and frontend work can integrate cleanly.
- As an integration developer, I want `RankedItem` to consistently include the fields required for rendering useful update cards.
- As an integration developer, I want category values to use the shared slugs so filtering, testing, and analytics remain consistent.
- As an integration developer, I want the PM docs to state non-goals clearly so I do not build auth, paid API support, vector search, or real scraping into the MVP by accident.

## Acceptance Themes

- Users should understand the feed after one quick pass.
- Users should be able to explain why surfaced items matter.
- Users should not see obvious duplicates.
- Users should trust that high-ranked items came from credible sources.
